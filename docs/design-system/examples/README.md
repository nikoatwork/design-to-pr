# Examples

Examples show how components combine into reviewable mockups.

## Current examples

### Design system gallery

```bash
npm run gallery
```

Use this to see components, foundations, guidance, props, and examples.

### Profile card

```bash
npm run dev profile-card
```

Use this as the starter mockup pattern.

## What makes a good example

- It is small enough to understand quickly.
- It uses existing components.
- It uses local mock data.
- It has a clear user, task, and tone.
- It can be screenshotted for review.
