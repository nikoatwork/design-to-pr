# Design Workflow

Use this repo as a shared sketching space for designers and coding agents.

## The working loop

1. **Look first.** Open the design system gallery and review available pieces.
2. **Choose a small target.** Ask for one card, panel, form, empty state, or screen section.
3. **Name the intent.** Say who it is for, what decision it helps with, and the desired tone.
4. **Review visually.** Ask the agent to preview or screenshot the mockup.
5. **Redirect in plain language.** Ask for changes like calmer, denser, more status-heavy, more editorial, more compact, or closer to the gallery.

## Good designer prompts

> Show me the design system gallery.

> Create an onboarding card for a new client admin using Button, Card, Badge, Input, Progress, and Switch.

> Make this mockup easier to scan. Keep the same content and components.

> Take screenshots of the current mockup for review.

## What to avoid

- Do not ask for a whole fake app when one focused screen is enough.
- Do not introduce real APIs or product data.
- Do not ask the agent to invent new visual primitives before checking the existing system.
- Do not judge only by code. Always review visually.
