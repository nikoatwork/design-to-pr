# Tones

Tones describe meaning before color.

Use tones consistently so designers, agents, and users understand why something has visual emphasis.

## Available starter tones

### Neutral

Default, quiet, historical, or low-priority information.

Use for: archived, saved, default, previously completed, secondary metadata.

### Success

Positive progress or completion.

Use for: ready, verified, complete, approved, sent, healthy.

### Warning

Something needs attention but is not yet critical.

Use for: pending, upcoming, incomplete, low balance, expiring soon.

### Danger

High-risk, destructive, failed, or urgent information.

Use for: delete, failed, overdue, blocked, error.

### Info

Guidance, help, updates, or system explanation.

Use for: tips, scheduled, updated, unread, setup guidance.

## Guidance

- Choose tone based on meaning, not decoration.
- Avoid using several high-emphasis tones in the same small area.
- Pair tone with clear copy. Color alone is not enough.
- Keep badges short.
- Use danger sparingly.

## Agent instruction

Before applying a colored Badge, message, or action, decide the tone in words. If the meaning is unclear, use neutral.
